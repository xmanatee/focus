# Fucus Design Context

## Users
iOS power-users fighting phone addiction or trying to do deep work. They live in the same world as Opal / One Sec / Jomo users but want something that commits them without scolding them.

## Brand Personality
**Warm. Confident. Alive.** The app is not a verdict, but it is honest — it makes the commitment felt without turning the user into the enemy of their own progress. Three words: **grounded, committed, alive**.

## Aesthetic Direction

### References
- **Headspace** — warm cream surfaces, rich saturated accents, rounded everything, generous spacing, playful confidence.
- **Duolingo** — fully-rounded buttons, vivid primary, big tappable targets, no-default feel.
- **Endel** — ambient craft, near-monochromatic palette, minimal motion.

### Anti-references (user-confirmed)
- **Freedom / clinical-corporate** — no flat blue, no enterprise chrome.
- **Apple Screen Time / Settings native** — no red-circle graphs, no default rounded-blue-icon grid.

### System theme (auto dark)
App responds to iOS `Appearance` — if the user's device is in Auto Dark Mode (Settings → Display & Brightness → Automatic), the app turns dark at sunset with the rest of the system. No in-app toggle; we follow the OS.

Light and dark palettes are defined as CSS variables in `global.css` and consumed via NativeWind's `darkMode: 'class'` (toggled in `app/_layout.tsx` from `useColorScheme()`).

### Typography
SF Pro only. Extreme weight contrast is the signature: `display-xl` Ultralight for numerals (the session timer), `display-md` Black for headlines. `label` is short-caps tracked, `body-md` is 17pt semibold (Duolingo-weighted body), `caption` is 14pt.

Tabular numerals on every digit via `fontVariant: ['tabular-nums']`.

### Roundness
Every surface reads rounded:
- Buttons → `rounded-full` (pill). Duolingo-level commitment.
- Inputs → `rounded-full` or `rounded-xl` (24px).
- Preset chips / day chips → `rounded-full`.
- Cards / rails → `rounded-2xl` (32px).
- Tab-bar glyphs → native SF Symbols (geometric consistency).

### Motion
Reanimated, exponential ease-out (no bounce). Press: `scale 0.97 @ 140ms`. Session start: `FadeIn 420ms`. Session end: `FadeOut 280ms`.

### Haptics (`expo-haptics` via `src/shared/design/haptics.ts`)
- `commit` — Rigid (start session, grant, save)
- `select` — selectionAsync (chip change, segmented toggle)
- `complete` — Success notification (session ends well)
- `abandon` — Soft impact (user aborts) — **never Error**

### Iconography
SF Symbols via `expo-symbols`, routed through `src/shared/components/Icon.tsx`. No emoji. No custom PNG icons. Anti-pattern script enforces this.

### Signature visual
The **countdown timer**: SF Pro Ultralight numerals at 72 pt, centered on the surface, with a 2 px signal-colored hairline underneath that depletes right-to-left. Full-screen takeover while a session is active — tab bar hides; no chrome except "Abandon."

## Design Principles
1. **Warm over cool.** Every neutral is warm-tinted. Pure black and white never appear.
2. **One accent, scarce.** Signal orange appears only on commit / active / primary-action surfaces.
3. **Rounded everything.** Pills for action, 2xl for surfaces. No sharp corners in the main flow.
4. **Typography carries hierarchy.** Ultralight + Black, not color tiers.
5. **Finish lines matter.** Every sensory moment has a defined end. No endless breathing loops.
6. **No guilt design.** Abandonment is soft. No red, no scolding, no "you failed" copy.
7. **System theme wins.** Fucus never decides for the user; iOS does.
